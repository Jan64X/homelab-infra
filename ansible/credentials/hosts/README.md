# Host-Specific Credentials Directory

This directory contains credentials generated for each managed host.

## Structure

Each host will have its own subdirectory named after the inventory hostname:

```
hosts/
  └── hostname/
      ├── sysadmin_pass.txt
      ├── salts/
      │   └── sysadmin_salt.txt
      └── [service-specific credentials]
```

## Service-Specific Credentials

Some hosts may have additional credential files depending on the services they run:

### Authentication & Access
- `grafana_admin_password.txt` - Grafana admin password
- `initial_password.txt` - Initial password for services like qBittorrent

### Backup Credentials
- NAS credentials for SMB-mounted backup shares are configured through group variables for the migrated homelab services.
- Service backup archives are written directly to the NAS and do not require per-host backup password files.

Other roles that still use a different backup mechanism may maintain their own service-specific credentials separately.

These are automatically generated by Ansible playbooks and stored securely. Do not delete unless performing a fresh installation.
